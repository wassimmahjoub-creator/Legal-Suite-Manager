BUG B1 — PALETTE CMD+K NE CHERCHE PAS DANS LES DONNÉES
═══════════════════════════════════════════════════════

COMPORTEMENT ACTUEL :
Ctrl+K → taper "عقار" → résultat "لا توجد نتائج" alors que "قضية عقار الزهراء" et "قضية شراكة" existent.

CORRECTION :

1. Vérifie l'endpoint GET /api/search?q=... — existe-t-il ? Renvoie-t-il les bonnes entités ?

2. Si l'endpoint n'existe pas ou est incomplet, le créer/compléter pour qu'il interroge en parallèle :
   - clients : LIKE sur name_ar, name_fr, tax_id
   - cases : LIKE sur case_number, title_ar, title_fr
   - invoices : LIKE sur invoice_number
   - documents : LIKE sur filename, title
   - courts : LIKE sur name_ar, name_fr
   - opponents : LIKE sur name_ar, name_fr

3. Renvoyer un format unifié { type, id, title, subtitle, icon, href, matchedField }.

4. Brancher la palette Cmd+K sur cet endpoint avec debounce 200ms.

5. Garder les actions rapides actuelles, mais en SOUS-section "إجراءات سريعة" après les résultats data.

6. Test : "عقار" doit ramener les dossiers contenant "عقار" dans leur titre.

BUG B7 — RETENUE À LA SOURCE NON APPLIQUÉE
═══════════════════════════════════════════════════════

COMPORTEMENT ACTUEL :
La colonne `ج.ض` (retenue à la source) existe en table factures mais est toujours à 0.000. Pourtant la spec T4 P1 prévoyait :
- Champ withholding_rate sur le client (0, 3, ou 5)
- Calcul automatique lors de l'émission de facture

CORRECTION :

1. Vérifie que la colonne `withholding_rate` existe bien sur la table `clients` (livrée en T3).

2. Pour les clients existants, mettre une migration douce :

   - L'utilisateur peut le passer à 3 ou 5 manuellement via la fiche client

3. Sur le formulaire de création/édition de client (page fiche client 360°), AJOUTER si absent :
   - Section "الجباية"
   - Champ "نسبة الخصم بالمصدر" : radio (0% / 3% / 5%)
   - Helper text : "تطبق على الأشخاص المعنويين الخاضعين للضريبة"

4. Sur le formulaire de facturation, dans la section récap :
   - Récupérer client.withholding_rate
   - Calculer withholding_amount = subtotal_ht * (withholding_rate / 100)
   - L'afficher en ligne "خصم بالمصدر (-X%)"
   - Soustraire du net_to_pay

   - Persister withholding_amount dans la colonne `ج.ض`
   - Persister withholding_rate appliqué (pour audit)
