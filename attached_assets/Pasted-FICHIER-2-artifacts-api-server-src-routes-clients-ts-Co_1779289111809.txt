Correction 2.1 — GET /clients : filtre deletedAt manquant sur la recherche + pagination
Trouve (lignes 25–37) :

router.get("/clients", async (req, res) => {
  const { search } = req.query as { search?: string };
  let clients;
  if (search) {
    clients = await db.select().from(clientsTable)
      .where(ilike(clientsTable.name, `%${search}%`));
  } else {
    clients = await db.select().from(clientsTable)
      .where(isNull(clientsTable.deletedAt))
      .orderBy(clientsTable.createdAt);
  }
  res.json(clients);
});
Remplace par :

router.get("/clients", async (req, res) => {
  const { search } = req.query as { search?: string };
  const page  = Math.max(0, parseInt((req.query.page  as string) ?? "0") || 0);
  const limit = Math.min(200, Math.max(1, parseInt((req.query.limit as string) ?? "50") || 50));

  const condition = search
    ? and(isNull(clientsTable.deletedAt), ilike(clientsTable.name, `%${search}%`))
    : isNull(clientsTable.deletedAt);

  const clients = await db.select().from(clientsTable)
    .where(condition)
    .orderBy(clientsTable.createdAt)
    .limit(limit)
    .offset(page * limit);

  res.json(clients);
});
Pourquoi : Le chemin avec search retournait les clients supprimés (soft-deleted) parce que le filtre isNull(deletedAt) n'était appliqué que dans l'autre branche else. Un client archivé apparaissait dans les résultats de recherche.

Remplace par :

router.get("/clients/:id/cases", async (req, res) => {
  const clientId = Number(req.params.id);

});
